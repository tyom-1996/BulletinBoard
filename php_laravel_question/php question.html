Ի՞նչ տեսակի զանգվածներ կան PHP -ում

1. Թվային - այն զանգվածները, որոնց տարրերն իրարից տարբերվում են թվային ինդեքսներով

2. Ասոցիատիվ  - այն զանգվածները, որոնց ինդեքսները բառեր են

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Բացատրել փոփոխականի տեսանելիության տիրույթի մակարդակները PHP -ում

Ծրագրավորման PHP լեզվում տվյալներն ունենում են տեսանելիության երեք մակարդակ:

  1. Լոկալ - ֆունկցիայի ներսում հայտարարված կամայական փոփոխական համարվում է լոկալ փոփոխական և այն տեսանելի է միայն ֆունկցիայի ներսում

  2. Գլոբալ - գլոբալ փոփոխականները նկարագրվում են ֆունկցիայից վեր, սակայն դրանք տեսանելի չեն ֆունկցիայի ներսում, քանի դեռ չի օգտագործվել global հրամանը
    Օրինակ:
    $x = 4;

    function F(){
       $x = 0;
    }

    F(); print $x; //4 ֆունկցիայի ներսում $x -ը գլոբալ չէ

    $x = 4; 

    function f(){
      global $x;
       $x=0;
    }

    F();

    print $x;     //0


Սուպեր գլոբալ - տեսանելիության այս տիրույթը բաց է և հասանելի է պրոյեկտի կամայական կետից

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ արդյունք կլինի էկրանին

$x = 4;

$y = &$x;

$x=0;

print $x . “ “ . $y;

Էկրանին կհայտնվի 0 0 տեքստը, քանի որ & նշանի կիրառման դեպքում $y փոփոխականի մեջ պահվում է ոչ միայն $x -ի արժեքը, այլ նաև հիշողության մեջ նրա հասցեն հետևաբար $y և $x -ը հղում են կատարում նույն հասցեին և նրանցից կամայական մեկի փոփոխությունը հանգեցնում է մյուսի փոփոխությանը.

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ է ֆունկցիայի պարամետրերի hint -ը

PHP -ում հնարավոր է սամանափակում տալ ֆունկցիային փոխանցվող պարամետրերին: օրինակ

function (Array $x){

}

Այս դեպքում ֆունկցիային հնարավոր է փոխանցել միայն զանգված, մյուս տիպերի դեպքում կստանանք Error:

PHP 5 -ում գործում են երկու տեսակի hint -եր

  1. Array

  2. Object

    1. Այդ թվում մեր կողմից ստեղծված կլասները

PHP 7 -ում կա նաև String, Int և այլն.

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ է հաստատունը PHP -ում

Հաստատունը որոշակի անվանում ունեցող տարածք է հիշողության մեջ, որը ի տարբերություն փոփոխականների երբեք չի փոխում իր արժեքը.

Հաստատունները նկարագրվում են define ֆունկցիայի օգնությամբ:

Օրինակ:

define("PI", 3.14);

print PI; //3.14

defined ֆունկցիայի օգնությամբ կարող ենք ստուգել արդյո՞ք այդ անունով հաստատուն արդեն նկարագրված չէր

Օրինակ

if( ! defined(“PI”) ){

define( “PI”, 3.14);

}

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ է ցույց տալիս __FILE__ հաստատունը

__FILE__ -ը վերադարձնում է տվյալ ֆայլի բացարձակ հասցեն

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ո՞րն է մեկ և կրկնակի ապաթարցների տարբերությունը

Կրկնակի ապաթարցների ներսում հնարավոր է օգտագործել փոփոխականներ, մյուսի դեպքում՝ ոչ:

$x = “Hayk”;

print  “<h1> Hello $x; </h1>”; // Hello Hayk

print  ‘<h1> Hello $x; </h1>’; // Hello $x

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ինչպե՞ս կարելի է տեղեկություն ստանալ, թե ինչ browser -ից է օգտվում User -ը

  1. Կարող ենք դիմել $_SERVER['HTTP_USER_AGENT']; փոփոխականին

  2. Կարող ենք օգտվել get_browser() ֆունկցիայից

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ է իրենից ներկայացնում սուպերգլոբալ զանգվածը

PHP -ում սուպերգլոբալ են այն զանգվածները, որոնք տեսանելի են պրոյեկտի բոլոր կետերից. 

  1. $_POST  - ցույց է տալիս post -ով եկած տվյալները

  2. $_GET - ցույց է տալիս get -ով եկած տվյալները

  3. $_REQUEST - ցույց է տալիս և post և get -ով եկած տվյալները

  4. $_FILES - ցույց է տալիս ֆայլերը, որոնք user -ը փորձում է upload անել

  5. $_SERVER - ցույց է տալիս ինֆորմացիա դեպի server եկած տվյալների մասին (օրինակ՝ ինչ browser -ից է եկել տվյալը, ip հասցեն, post, թե get և այլն) 

   6. $_SESSION - պահպանում է session -ի տվյալները 

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ տարբերություն կա POST/GET -ի միջև

POST -ը և GET -ը դեպի սերվեր տվյալներ ուղարկելու եղանակներ են:

  1. GET - տրամադրում է հիշողության ավելի փոքր տարածք,  ուղարկվող տվյալները բացահայտ երևում են URL -ում, օրինակ՝ ?name=Hayk&password=111, անվտանգ չէ

   2. POST - տրամադրում է հիշողության ավելի մեծ տարածք, տվյալները չեն     երևում browser -ում, հետևաբար անվտանգությունն ավելի բարձր է այստեղ.

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ տարբերություն կա isset և empty, unset ֆունկցիաների միջև

  1. Isset - ստուգում է արդյո՞ք փոփոխականը հայտարարված է եղել թե ոչ

  2. Empty - ստուգում է արդյո՞ք դատարկ է տվյալ փոփոխականը (զանգվածը)

  3. Unset - ջնջում է տվյալ փոփոխականը
    $x = 4;
    unset($x);
    print $x; //error

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչպես կարելի է ստեղծել ֆայլեր և թղթապանակներ PHP -ում

Ֆայլեր ստեղծելու համար օգտվում ենք fopen ֆունկցիայից

Օրինակ՝

 $f = fopen(“erg.mp3”, “w”); //ստեղծվում է erg.mp3 ֆայլը

w -ն նշանակում է, որ ֆայլը բացվում է գրելու ռեժիմում, այսինքն նրա նկատմամբ հնարավոր է կիրառել fwrite ֆունկցիան.

w - ի փոխարեն կարող է լինել նաև

  • r - կարդալու ռեժիմ

  • r+ - կարդալու/գրելու ռեժիմ (ցուցիչը տեղադրում է ֆայլի սկզբում)

  • w+ - գրելու/կարդալու ռեժիմ (ցուցիչը տեղադրում է ֆայլի վերջում՝ ջնջելով պարունակությունը)

  • a - գրելու այս ռեժիմում թույլ է տրվում ավելացնել ֆայլին ինֆորմացիա առանց նախկինը ջնջելու

  • a+  - գրելու/կարդալու այս ռեժիմում նախկին պարունակությունը չի ջնջվում 

  • x  - ստեղծում է նոր ֆայլ գրելու ռեժիմում, վերադարձնում false, եթե ֆայլն արդեն կար

  • x+  - ստեղծում է նոր ֆայլ գրելու/կարդալու ռեժիմում, վերադարձնում false, եթե ֆայլն արդեն գոյություն ունի: 

Թղթապանակ ստեղծելու համար օգտվում ենք mkdir ֆունկցիայից.

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ է SESSION -ը

Session -ը հիշողության տարածք է սերվերում, որտեղ հնարավոր է պահպանել տարբեր ծավալի ինֆորմացիա. Session -ը ավտոմատ կերպով ջնջվում է երբ Browser -ը փակվում է: 

Session -ի հետ աշխատանքը սկսվում է session_start() ֆունկցիայով, այնուհետև կարող ենք տարրեր ավելացնել $_SESSION սուպերգլոբալին. Session -ի տարրեր հնարավոր է ջնջել unset ֆունկցիայիով, իսկ ամբողջ սեսսիան կարելի է հեռացնել session_destroy ֆունկցիայի օգնությամբ:

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ է COOKIE -ն

Cookie -ն հիշողության տարածք է, որը պահպանվում է Browser -ում:

Cookie -ն ստեղծվում է setcookie ֆունկցիայի օգնությամբ: 

setcookie(x,y,z)

X - անուն

Y - արժեք

Z - տևողություն, օրինակ՝ time() + 86400 -ի դեպքում կպահպանվի մեկ օր

Cookie -ն ջնջվում է setcookie ֆունկցիայիով, սակայն տևողության համար ընտրվում է բացասական արժեք: օրինակ՝ time() - 86400

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ ֆիլտրեր կան PHP -ում

php -ում filter_var ֆունկցիայի օգնությամբ կարելի է վալիդացիայի ենթարկել ինֆորմացիան: 

Օրինակ՝

  1. Ստուգելու համար արդյո՞ք $email -ը ճիշտ է գրված կարելի է գրել
    if( filter_var($email, FILTER_VALIDATE_EMAIL) === true)

  2. Կան նաև այլ հաստատուններ

    1. FILTER_VALIDATE_INT

    2. FILTER_VALIDATE_FLOAT

    3. FILTER_VALIDATE_STRING

Filter_var ֆունկցիան հնարավորություն է տալիս նաև ֆիլտրել տրված փոփոխականը:

Օրինակ $email -ի միջից բոլոր ավելորդ սիմվոլները (օրինակ / ) կարելի է հեռացնել այսպես

  3. Protected - մասնակի  հասանելիություն, դաշտեր, որոնք հասանելի են կլասի ներսում և ժառանգ կլասների ներսում:

Երկու դաշտեր հնարավոր է կապել foreign key -ով, եթե.

  1. Այդ դաշտերն ունեն տվյալների նույն տիպը

  2. Եթե աղյուսակներից մեկում լրացված չեն այնպիսի տվյալներ,  որոնք գոյություն չունեն մյուս աղյուսակում

Ի՞նչ սահմանափակումներ կարելի է տալ Foreign Key ավելացնելիս

ON DELETE / ON UPDATE. Ենթադրենք ունենք A և B աղյուսակներ, foreign key -ը գտնվում է B -ում:

  1. CASCADE  - A -ից տվյալ ջնջելիս ջնջվում են նրան համապատասխան տվյալները B -ից

  2. DO NOTHING - A -ից տվյալ ջնջելիս B -ում համապատասխան տվյալները պահպանվում են 

  3. SET NULL - A -ից տվյալ ջնջելիս նրան համապատասխան տվյալները B -ում ստանում են Null արժեք

  4. RESTRICT - A -ից անհնար է տվյալներ ջնջել, քանի դեռ կան կապակցված տվյալներ B -ում

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան
Ի՞նչ տեսակի join -եր կան

  1. Inner join - A և B աղյուսակներից վերցնում է միայն կապակցված և հստակ համապատասխանության մեջ դրված տվյալները

  2. Left join - վերցնում է A աղյուսակն ամբողջությամբ, նրան համապատասխանության մեջ է դնում B -ի տվյալները. Այն դաշտերը, որոնք համապատասխան արժեքներ չեն ունենում դառնում են Null

  3. Right join - վերցնում է B աղյուսակն ամբողջությամբ, նրան համապատասխանության մեջ է դնում A -ի տվյալները. Այն դաշտերը, որոնք համապատասխան արժեքներ չեն ունենում դառնում են Null

Պատասխանող՝ Ուսումնական կենտրոնի ղեկավար, Արմեն Մարջինյան

Union հրամանը թույլ է տալիս միավորել մի քանի select-ների արդյունքներ, եթե այդ սելեկտներում սյուների քանակը  և տվյալների տիպը նույնն է: union -ը ազատվում է կրկնվող տվյալներից, իսկ union all -ը թույլ տալիս դրանց կիրառումը:

Ի՞նչ է view-ն

View -ն տվյալների բազայի օբյեկտ է, որի ներսում հնարավոր է պահել select -ի արդյունք: Ենթադրենք A,B,C աղյուսակներն իրար կապելուց և x1,x2,x3 պայմանները գրելու արդյունքում ստացվել է ինչ-որ M արդյունք: Այդ արդյունքը կարելի է պահպանել view -ի մեջ

Create view ‘myview’ as select M

Այս դեպքում M անվան տակ կպահպանվի ամբողջ արդյունքը, որը հնարավոր է կիրառել այլ դիրքերում: Օրինակ select * from MyView where id > 4

Չնայած նրան, որ MySQL -ը ռելացիոն բազա է և կապերի օգնությամբ կրճատվում է կեղծ տվյալներ ունենալու հավանականությունը, այնուամենայնիվ տվյալները հնարավոր է ավելացնել նաև ձեռքով, ինչը կարող է հանգեցնել ոչ լրիվ տվյալների:

Օրինակ սոցիալական ցանցում առկա են friends (u1,u2) և namakner(u1,u2) աղյուսակները:

Հնարավոր է այնպես անել, որպեսզի Namakner աղյուսակում լինի ենթադրենք u1=8, u2 = 32 տվյալ, այսինքն 8-րդ օգտագործողը նամակ է գրել 32 -րդին, սակայն սա կհամարվի սխալ, եթե Friends աղյուսակում նրանք ընկերներ չեն: Այսինքն կստացվի, որ ոչ friend-ները կարող են իրար նամակ գրել: Նմանատիպ իրավիճակներից խուսափելու համար ապահովում են տվյալների բազաների ամբողջականությունը, որը տեղի է ունենում trigger -ների օգնությամբ:
